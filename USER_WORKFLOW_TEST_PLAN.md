# 🧪 使用者日常操作測試計畫 (User Workflow Test Plan)

本文件定義了使用者在 `Legado Reader` 中的核心操作流程，並作為程式碼邏輯驗證的基準。

---

## 📂 流程一：書源準備 (Source Preparation)
使用者在開始閱讀前，通常需要匯入書源。
- [✅] **1.1 從 URL 匯入書源**：輸入一個 `.json` 或 `.txt` 的網路連結並成功解析。
- [✅] **1.2 從二維碼匯入**：啟動相機掃描並獲取書源資料。 (已補強 legado:// 協定支援)
- [✅] **1.3 書源管理**：啟用/禁用特定書源，並能進入編輯介面查看規則。
- [✅] **1.4 分組篩選**：在書源列表透過上方標籤（如：出版、精選）進行過濾。

## 📂 流程二：尋找書籍 (Book Discovery)
- [✅] **2.1 全域搜尋**：輸入關鍵字，同時從多個已啟用的書源獲取結果。 (具備併發控制與聚合邏輯)
- [✅] **2.2 發現頁面瀏覽**：選擇一個書源，進入其「發現」分類（如：玄幻、都市）查看熱門書籍。
- [✅] **2.3 書籍詳情與加入書架**：查看章節列表，點擊「加入書架」。

## 📂 流程三：本地書籍匯入 (Local Book Import)
- [✅] **3.1 檔案選擇器匯入**：從系統檔案 App 選擇一個 `EPUB` 或 `TXT` 檔案。 (已優化大檔案分批寫入)
- [⚠️] **3.2 智能掃描**：自動掃描手機內部的書籍檔案。 (代碼中暫未發現自動全盤掃描邏輯，目前依賴手動選取)
- [✅] **3.3 本地書籍閱讀**：確保本地檔案能正確分頁並載入目錄。

## 📂 流程四：閱讀體驗 (Reading Experience)
這是 App 的核心場景。
- [✅] **4.1 章節載入與緩存**：點擊章節，確保內容能從網路獲取並持久化至資料庫。
- [✅] **4.2 樣式調整**：調整字體大小、行間距、佈景主題（深/淺色）。
- [✅] **4.3 字體管理**：匯入自訂 `.ttf` 字體並成功套用於閱讀介面。
- [✅] **4.4 字典查詞**：長按選取文字，彈出字典對話框並顯示解析後的結果。 (已優化標籤替換邏輯)
- [✅] **4.5 目錄導航**：從左側或下方喚出目錄，快速跳轉章節。

## 📂 流程五：同步與備份 (Sync & Backup)
- [✅] **5.1 WebDAV 設定**：輸入伺服器位址與帳密，測試連接。
- [✅] **5.2 全量備份**：將書架、書源、設定打包並上傳至 WebDAV。 (已補全所有 DAO 資料)
- [✅] **5.3 進度同步**：更換設備或重裝後，能自動/手動從雲端恢復書籍進度。
